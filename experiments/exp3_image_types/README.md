# Experiment 3: Image Type Analysis

## Objective

Test the StarVector-1B model with different types of images to evaluate whether it generates valid and visually acceptable SVGs across varying image complexities.

This experiment runs 5 independent generations per image to assess model consistency and output quality.

## Configuration

I tried to use the baseline config, that has max_length=4000, but it was not enough, so I changed to 6000. Configuration:
- `temperature=1.5`
- `max_length=6000`
- `repetition_penalty=3.1`

## How to Run

### Single Generation
```bash
python experiments/exp3_image_types/exp3_image_types.py \
  --images assets/examples/sample-0.png \
  --output-dir experiments/exp3_image_types/results
```

### Multiple Images
```bash
python experiments/exp3_image_types/exp3_image_types.py \
  --images assets/examples/sample-*.png \
  --output-dir experiments/exp3_image_types/results
```

### Multiple Generations (Automated)
To run multiple independent generations for consistency analysis:
```bash
conda activate starvector
bash experiments/exp3_image_types/run_multiple_generations.sh
```

This script generates 5 independent runs and saves them to `results1/` through `results5/`.

### Compare Results
After running multiple generations:
```bash
python experiments/exp3_image_types/analyze_multiple_runs.py
```
